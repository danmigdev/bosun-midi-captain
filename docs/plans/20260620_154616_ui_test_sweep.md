## Bugs found AND fixed (source; need rebuild to land in portable)
- BUG-1 (user-reported): combobox next to "+ add message" used browser-default
  styling (white bg, gray border, no padding, font NOT scaled with UI zoom),
  inconsistent with every other dropdown. Root: `.addmsg select` missing from
  the shared select rule in PatchEditor.svelte:605. FIX: added `.addmsg select`
  to that selector. Verified live via CSS injection (14_ before / 15_ after).
- BUG-2: hardcoded ITALIAN, Ampero-specific string in MidiLearn.svelte:117
  ("Cambia patch sull'Ampero... apparira qui") shown while learning, even on a
  Kemper profile. Violates English-only content rule. FIX: replaced with a
  device-neutral English "Listening..." hint.

- BUG-3 (user-reported): false "A pedal without Bosun firmware is connected."
  on a flashed pedal. Root: pollForUnflashedPedal (App.svelte) prompts after a
  SINGLE failed autoConnect in the stockSerial branch; a freshly-plugged pedal
  still booting / with its data CDC not yet enumerated fails that one probe.
  Also no re-entrancy guard, so overlapping ~9s probes contend for the port.
  FIX: re-entrancy guard (_pollInFlight) + consecutive-miss debounce
  (_stockSerialMisses, 3 retries) before prompting; unambiguous drive/bootloader
  states still prompt immediately.
